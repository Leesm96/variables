# MVP 1.0 Spec

## 입력 필드
| 필드 | 타입 | 설명 | 예시 |
| --- | --- | --- | --- |
| 도징 | number | 포터 필터에 담는 원두량(g). | 18.5 |
| 추출량 | number | 추출 후 컵에 담긴 에스프레소 양(g). | 38 |
| 시간 | number | 추출에 소요된 시간(초). | 27 |
| 온도 | number | 추출 시 설정한 물 온도(°C). | 93 |
| 맛 상태 | string | 현재 컵 프로파일을 간단히 서술(예: 산미 강함, 텁텁함). | "산미가 날카롭고 단맛이 부족" |
| 로스팅일 | string | 로스팅 완료 후 경과 일자 혹은 날짜(ISO 8601). | "2024-06-01" |
| 매장 온도 | number | 추출 환경의 실내 온도(°C). | 25 |
| TDS | number | 굴절계로 측정한 총 용존 고형물(%). | 9.8 |
| 로스팅 단계 | string | 라이트/미디엄/다크 등 로스팅 레벨. | "Medium" |

## 출력 필드
| 필드 | 타입 | 설명 | 예시 |
| --- | --- | --- | --- |
| TOP3 조정 요소 | string[] | 가장 큰 영향력을 가진 조정 포인트 3가지. | ["분쇄도 미세 조정", "추출 시간 +2s", "도징 +0.2g"] |
| 수치 조정 | object | 각 요소별 권장 변경값(양수/음수 모두 허용). | {"도징": "+0.2g", "추출량": "-2g"} |
| 맛 변화 예상 | string | 조정 이후 예상되는 맛 변화 설명. | "단맛이 올라가고 산미가 부드러워집니다." |
| 상황별 힌트 | string[] | 특정 상황(과소/과다 추출 등)에 따라 적용할 팁. | ["과다 추출 시 추출량 -2g", "산미 과하면 분쇄 굵게"] |
| 피드백 구조 | object | 바리스타가 결과를 기록할 피드백 항목 구조. | {"세션메모": "문장", "다음 시도": "숫자/메모"} |

## 공용 JSON 계약 예시
프런트엔드와 백엔드가 공유하는 요청/응답 형식 예시는 아래와 같다.

### 요청(Request)
```json
{
  "dose_g": 18.5,
  "yield_g": 38,
  "brew_time_s": 27,
  "temperature_c": 93,
  "tasting_note": "산미가 날카롭고 단맛이 부족",
  "roast_date": "2024-06-01",
  "shop_temp_c": 25,
  "tds_pct": 9.8,
  "roast_level": "Medium"
}
```

### 응답(Response)
```json
{
  "top3_adjustments": ["분쇄도 미세 조정", "추출 시간 +2s", "도징 +0.2g"],
  "numeric_tweaks": {
    "dose_g": "+0.2g",
    "yield_g": "-2g",
    "temperature_c": "+0.5°C"
  },
  "expected_flavor_change": "단맛이 올라가고 산미가 부드러워집니다.",
  "contextual_hints": [
    "과다 추출 시 추출량을 2g 줄이세요.",
    "산미가 과하면 분쇄도를 한 단계 굵게 조정하세요."
  ],
  "feedback_schema": {
    "session_note": "string",
    "next_try": "string",
    "taste_rating": "number"
  }
}
```
